## 主張: 構築分類は明確に分離していない

古典分類:
| 分類 | コア発想 |
|---|---|
| 対面 | 場の1対1で勝ち、繰り出し勝ちは捨てる |
| サイクル | 繰り出し有利のポケで回す |
| 受けループ | 防御寄り3-4体で粘る |
| バランス | 上記の混合 |

**現代の解釈**: これらの境界は実戦では薄い。
- 「対面構築」と呼んでいる並びでも実態はサイクル要素を持つ
- 「サイクル」だが対面性能の高いコマもある
- 「受けループ」が崩しエースを 1 体採用するのは標準

→ **採用基準**: 「役割が明確に説明できるか」が分類より重要。

## 役割理論の現代解釈

| 役割 | 役割定義 | 例（環境シェア順、世代問わず） |
|---|---|---|
| 起点作り | ステロ・積み・ステ補助で後続を強化 | サンダー、ランドロス、エースバーン等 |
| クッション | 削れて後続に繋ぐ | 再生力ヌオー、ピリピリヤドキング等 |
| ストッパー | 相手の積み・崩しを止める | 不意ガブ、こだわりトレース等 |
| エース（決定力） | 終盤の詰め筋 | 鉢巻ウーラオス、特化ポリ2 等 |
| 抜きエース | 速度補助で全抜き | 龍舞ガブ、剣舞ハッサム等 |
| 起点解除 | 相手の積み駒を即座にKO | てんねん、シャドダ等 |
| 受け（潰し主体） | タイプ受けで起点化 | ヒードラン、ガッサ等 |
| 受け（混合） | 削り役 + 補助 | ドオー、ヘイラッシャ等 |

## 採用理由言語化（必須）

各ポケに対して以下のテンプレで言語化する:

```
{ポケ名}:
  役割: {主要役割}
  対象: {仮想敵2-3体}
  採用理由: {なぜこのコマでないとダメか}
  代替: {他に役割を担えるコマ}
  弱点: {このコマだけだと崩れる軸}
```

例:
```
ガブリアス:
  役割: ストッパー / 起点作り（ステロ）
  対象: 龍舞カイリュー、ジャラランガ、テツノカイナ
  採用理由: A種族値130+S102は環境最速級ストッパー、ステロ + 龍舞 + 鉢巻3型可変
  代替: ランドロス（特殊耐性は厚いが鉢巻型では火力不足）
  弱点: 氷4倍タイプ、エルフブースト Iron Crown 受け切り不可
```

## 使用率上位への対応 (HARD RULE)

> 環境シェア top 10 に対して、自軍に **対応駒を最低2体** 用意する。

理由: 1 体で受けると相手のテラ / 持ち物変動で崩れる確率が高い。
2体で「対面 + サイクル」または「役割破壊保険」のセットを組む。

## 環境最適に型を可変

採用理由（ポケモン自体）は**不変**、技 / 持ち物 / 振り は**可変**。

| 例 | 採用理由 | 型バリエーション |
|---|---|---|
| ガブリアス | 環境ストッパー / 龍 / 地面 | 鉢巻 / 龍舞剣舞 / ステロ / 珠スカーフ |
| ヘイラッシャ | 物理クッション / ドラゴン受け | 残飯 / 食べ残し / 半減実 / オボン |
| ハバタクカミ | 特殊エース / 起点解除 | ブースト / スカーフ / こだわりメガネ / 残飯 |

## 現代構築の標準フロー

1. **コンセプト**: 中核2-3体（決まったコンボ/コアロール）を決める
2. **環境スキャン**: 直近1ヶ月の使用率TOP20 を `scripts/parse_usage.py` で取得
3. **対応駒配置**: TOP10 に対して2体ずつカバー
4. **エース 1-2体**: 終盤詰めの決定力枠
5. **起点作り 1体**: ステロ or 補助技
6. **誤魔化し / フェイク**: 詰みを避ける応急処置1体（オプション）

## 「現代構築」は古典役割の融合

役割破壊耐性 / 役割対象の二重化 / 採用理由の言語化 — この3軸が揃えば、古典分類のどれを名乗っても問題ない。
スキルとしては「○○構築」とラベリングせず、**役割言語**で説明する。

## このドキュメントの使い方

- T2 (構築 / サイクル) 質問が来たら本ドキュメントの「採用理由言語化テンプレ」で構造化
- 質問者が「対面構築教えて」と言っても、役割テンプレで答える（分類論争に引きずられない）
- 環境最適化が要求されたら直近 stats/ JSON を参照して TOP10 対応をチェック
