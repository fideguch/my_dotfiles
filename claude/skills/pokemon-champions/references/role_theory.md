# 役割理論 — 古典の整理

> 役割理論はポケモン日本コミュニティ発祥（2000年代）の構築論。
> 現代解釈は `modern_team_building.md` 参照。

## 古典の8役割

| 役割 | 定義 | 持ち物例 | 技例 |
|---|---|---|---|
| アタッカー | 中盤〜終盤の打点 | 命の珠、こだわり系 | 高威力タイプ一致技 |
| エース | 確実に詰める決定力 | 命の珠、こだわりメガネ | 高威力連発 |
| サイクル要員 | 繰り出し→打ち→引きで回す | オボン、半減実、再生力 | 自然回復 / 中威力 |
| 起点作り | ステロ / 補助技で後続強化 | 残飯、メンタルハーブ | ステロ、欠伸、補助 |
| クッション | 削れて後続に繋ぐ役 | 食べ残し、再生 | 自然回復、ドレインキッス |
| ストッパー | 相手の積み駒/崩しを止める | こだわりスカーフ | 不意打ち / 先制技 |
| 抜きエース | 速度上昇で全抜き | 残飯、ラム実 | 龍の舞、剣の舞 |
| 積み駒 | 自身が積んで詰める | リリバの実、ラム実 | 龍舞、剣舞、瞑想 |

## 役割破壊と保険

役割破壊: ある役割を担うコマが、別のタイプ技で崩される現象。
例: 受け側ヌオーが Grass Knot で崩される。

役割破壊保険: 役割破壊コマに対して、自軍の別コマが処理できる構造。
例: ヌオーが Grass Knot で崩されたら、ヘイラッシャが受け切る。

```
[相手] → [自軍 役割] → 役割破壊技 → [自軍 別コマ 保険]
```

## 役割対象表の作り方

NxM マトリクスで各駒対1駒の関係を埋める。

```markdown
| 自軍 \ 相手 | A | B | C |
|---|---|---|---|
| 自1 | ○ | △ | × |
| 自2 | △ | ○ | △ |
| 自3 | × | △ | ○ |
```

凡例:
- `○` 確定有利（受け切り、または上から殴る）
- `△` 五分（互角、もしくはサイクル次第）
- `×` 確定不利（即死、もしくは交代不可）

最低条件: 全相手列に `○` が 1 個以上、全自軍行に `×` が 0 個。

## 古典 vs 現代の違い

| 軸 | 古典 | 現代 |
|---|---|---|
| 役割 | 1コマ 1役割が基本 | 1コマ多役割（型変動） |
| 環境追従 | 半年単位で更新 | 月次で型変動 |
| ラベリング | 「アタッカー」「エース」明示 | 役割言語で説明（ラベル軽視） |
| 受け | タイプ受け中心 | 状態異常 + タイプ + 削り混合 |
| 攻撃 | 単純高火力 | テラ / Z で型急変 |

## 役割理論はまだ有効

「分類は明確じゃない」と言いつつ、役割言語そのものは現役。
古典: 「これはアタッカーです」
現代: 「これは ガブストッパー兼ステロ撒きです」
→ 構造は同じ。表現が機能ベース。

## このドキュメントの使い方

T2 質問で構築相談を受けたら:
1. 自軍 6 体（または仮想 6 体）に対して役割を割り当て
2. 役割対象表を作成
3. 全列 `○` が埋まっているか確認
4. 全行 `×` が 0 か確認
5. 不足があれば代替コマ提案
