# 構成MDの書式見本

このファイルは、ページ一覧→ワイヤーフレーム→内容の順で構成MDを書くための最小例です。実案件では案件固有の文章・URL・プロンプトに置き換えます。

## ページ一覧

1. キックオフ｜本日の流れと到達点
2. レクチャー回｜今回のテーマと再現手順

## ページ1｜本日の流れと到達点

**ワイヤーフレーム**

```text
章ラベル
ページ題
要旨（1〜2文）
水平罫線
本日の流れを説明する散文
到達点を示す【決定事項】枠
フッター：資料名・回／1 / 2
```

**内容**

- **要旨**：本日は現状を確認し、次回までに再現できる到達点を合意します。
- **本文**：この資料は、会議中の説明順に沿って、背景、確認事項、次の行動をまとめたものです。会議後は資料だけで決定内容と再現手順を読み返せます。
- **【決定事項】**：今回の到達点は〔クライアント固有の到達点〕とする。
- **URL**：初出のツールは目的画面の直URLを全文で記載する。最終ページの「本資料のURL一覧」にも再掲する。

## ページ2｜今回のテーマと再現手順

**ワイヤーフレーム**

```text
章ラベル
ページ題
要旨（1〜2文）
水平罫線
今回やることの説明（散文）
手順の番号付き列挙
プロンプト見出し・使い方
プロンプトブロック
完了確認
フッター：資料名・回／2 / 2
```

**内容**

- **要旨**：今回の操作を資料だけで再現し、〔確認する成果〕まで到達できるようにします。
- **本文**：まず〔用意するもの〕を確認し、次に〔目的画面〕を開きます。操作の理由を説明してから手順へ進み、最後に自分の環境で同じ結果になるかを確認します。
- **手順**：1. 〔手順1〕。 2. 〔手順2〕。 3. 〔完了条件〕を確認する。
- **プロンプト1：（〔目的〕）**：〔貼り付け先〕に貼る。貼る前に〔準備する情報〕を用意する。

```text
〔ここに構成MDの本文として確定したプロンプトを一字一句記載する〕
```

- **完了確認**：〔成果物または画面状態〕になっていることを確認する。記入欄は作らず、未完了の場合は会議後に構成MDを更新する。
